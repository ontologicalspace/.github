# Support

## How to get help

| You want to… | Use |
|--------------|-----|
| Report a **bug** | [Issue tracker → Bug Report][bug] |
| Request a **feature** | [Issue tracker → Feature Request][feat] |
| Ask a **usage / setup** question | [Issue tracker → Question][q] |
| Report an **evidence / invariant** problem | [Issue tracker → Paper/Evidence][ev] |
| Report a **security vulnerability** | See [`SECURITY.md`](SECURITY.md) — **never** a public issue |

[bug]: https://github.com/ontologicalspace/osp/issues/new/choose
[feat]: https://github.com/ontologicalspace/osp/issues/new/choose
[q]: https://github.com/ontologicalspace/osp/issues/new/choose
[ev]: https://github.com/ontologicalspace/osp/issues/new/choose

> GitHub Discussions are not yet enabled. Until they are, usage questions go through the
> *Question* issue form. This keeps a single discoverable channel.

## Before filing

- Search existing issues to avoid duplicates.
- For bugs and questions, include the OSP version/commit and the interface you used
  (CLI / MCP / Desktop / library).
- Read the relevant paper or spec section — OSP is evidence-driven, and many answers
  point to the invariant or proof that motivates a behavior.
